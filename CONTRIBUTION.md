# Contribution Guide

Contributions are welcome if they help make this repository more useful for security interview preparation.

## What You Can Add

- Security engineering interview questions
- Security research interview questions
- Interview experiences and formats
- Secure coding or code review examples
- Threat modeling and architecture questions
- Short explanations, answers, or study notes

## Writing Style

Keep the writing clear, casual, and practical. The goal is to make the notes easy to read, easy to update, and useful for interview preparation.

## Before Contributing

- Do not include confidential company material.
- Do not include private interviewer names or personal information.
- Keep examples educational.
- Add enough context so someone else can understand the question or note.

## Suggested Format

For interview experiences, try to include:

- Company or role
- Interview round or format
- Topics covered
- Questions asked
- Short answers or notes, if useful
